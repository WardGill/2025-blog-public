# 导言

## 关于本系列笔记

本系列笔记主要面向**蓝桥杯 C++ 比赛**，因为在实际比赛中会大量用到 STL（Standard Template Library，标准模板库）中的内容。

然而学校的课程安排通常不会深入讲解 STL，因此本笔记：

- **不追求原理讲解**：以理解和使用为首要目标
- **注重接口与函数**：提供方便快捷的查阅手册
- **结合实际应用**：展示竞赛中的典型用法

---

## 学习建议

1. **配合视频学习**：建议结合黑马程序员 C++ 课程中的 STL 部分学习
2. **理解数据结构**：对于没学过数据结构的同学，视频中也会简要介绍相应的数据结构，方便理解 STL 的设计原理
3. **动手实践**：每个容器后都有相应的例题练习，来自力扣或其他算法平台

---

## 蓝桥杯与 STL

蓝桥杯的赛制特点：

> 只要保证每道题目能够复现题目的过程，即使没有全部通过测试，但每道题都通过了一部分，仍有较大概率进国赛。

**STL 的优势**：
- 让代码编写过程更加简便
- 减少手动实现数据结构的繁琐工作
- 专注于算法逻辑而非底层细节
- 提高代码的正确性和稳定性

## STL 三大组件

| 组件 | 说明 |
|------|------|
| **容器** | 管理数据的集合，如 vector、map、set |
| **算法** | 处理数据的操作，如 sort、find、transform |
| **迭代器** | 连接容器与算法的桥梁，用于遍历和访问容器元素 |

---

## 注意事项

1. **命名空间**：STL 位于 `std` 命名空间中，使用时可以 `using namespace std;`
2. **头文件**：不同容器需要包含对应的头文件（如 `<vector>`, `<string>`）
3. **时间复杂度**：竞赛中要关注操作的时间复杂度，选择合适的容器
4. **迭代器失效**：插入和删除操作可能导致迭代器失效，使用时要小心
